# ①課題名
マーケティングプロンプトメーカーver4
〜外部マテリアル出力ver〜

## ②課題内容（どんな作品か）
- マーケティングの業務に活用するためのプロンプトを作成するためのWebアプリ

- 以下、挙動
- ①新規登録画面でメールアドレスとパスワードを入力 ※ユーザー毎にユニークなIDを付与
- ②フォームに自社サービスの情報（概要、ブランド、ターゲット）を入力 ※ここではまだプロンプト品質は粗い
- ③入力情報をGeminiへ飛ばし、プロンプトを精度高く翻訳
- ④Geminiからプロンプトを受け取り、Webアプリ側に表示
- ⑤FirebaseのFireStoreへプロンプトを保存 ※ユニークID単位（=ユーザー単位）でプロンプトが保存される
- ⑥Geminiから受けとったプロンプト結果をPHPへ渡し、外部マテリアルへ出力 ⭐️新規追加

## ⑤工夫した点・こだわった点
- 卒業プロダクトに実装予定の「入力内容を生成AIに翻訳させ出力する」のプロトタイプ第二弾
- アンケートのコア機能は「情報の入力＆結果の出力」であるため、「情報の入力=前回作成したGeminiから受け取ったプロンプト結果」と定義し、それを外部マテリアル（data.txt）に出力する形とした
- 過去制作アプリはXAMMPを経由してなかったため、まずは丸ごとXAMMPで開発を行い、さくらレンタルサーバーへ移動した

## ⑥難しかった点・次回トライしたいこと（又は機能）
- 実現にあたりMacの権限によるブロックが3度発生し、ターミナルを経由し本アプリに限定して権限を全て解放する必要があったこと
- PHPの概念がフロントエンドと異なる点も多く、フロントエンドとの接続の理解に苦戦した（例：PHPはフロントエンドのコードをそのままでは読み込みできない、など）
- 外部マテリアルの出力をPDFやGoogleスライドに行いたかったが時間が足りず断念
