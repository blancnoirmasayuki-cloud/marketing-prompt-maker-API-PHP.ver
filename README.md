# ①課題名
マーケティングプロンプトメーカーver5
〜PHPによるデータベース処理機能追加〜

## ②課題内容（どんな作品か）
- マーケティングの業務に活用するためのプロンプトを作成するためのWebアプリ

- 以下、挙動
- ①新規登録画面でメールアドレスとパスワードを入力 ※ユーザー毎にユニークなIDを付与
- ②フォームに自社サービスの情報（概要、ブランド、ターゲット）を入力 ※ここではまだプロンプト品質は粗い
- ③入力情報をGeminiへ飛ばし、プロンプトを精度高く翻訳
- ④Geminiからプロンプトを受け取り、Webアプリ側に表示
- ⑤FirebaseのFireStoreへプロンプトを保存 ※ユニークID単位（=ユーザー単位）でプロンプトが保存される
- ⑥Geminiから受けとったプロンプト結果をPHPへ渡し、メモへ出力
- ⑦Geminiから受けとったプロンプト結果をPHPで操作し、データベースへ保存 ⭐️新規追加

- 過去に作ったアプリにPHPとデータベースへの保存機能を追加したこと

## ⑥難しかった点・次回トライしたいこと（又は機能）
- コードというより、フロントエンドに比べてエラーの原因が分かりにくく解消するのに手間取った
- PHPのコードではあるが、PHPを動かすためにJavaScriptなども少し調整する必要があったこと
- Geminiの使いすぎで強制待機させられたこと
